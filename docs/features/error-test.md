# 錯誤測試 (Error Test)

## 功能介紹

錯誤測試功能是開發環境專用的工具，用於測試應用程式的錯誤處理機制和 Sentry 錯誤追蹤整合。本功能模組提供了手動觸發錯誤的能力，幫助開發者驗證錯誤捕獲、錯誤報告和 Sentry 整合是否正常運作。此功能僅在開發環境中啟用，生產環境中不會顯示，確保不會影響一般使用者的體驗。

## UX 路徑

### UX0032: 觸發測試錯誤

**觸發條件**：開發環境中，`ENABLE_ERROR_TEST` 為 true，使用者進入錯誤測試頁面

**操作步驟**：
1. 開發環境中，應用程式已啟用錯誤測試功能
2. 使用者導航到錯誤測試頁面（通過路由或直接訪問）
3. 錯誤測試頁面顯示錯誤觸發按鈕
4. 點擊錯誤觸發按鈕
5. 系統觸發測試錯誤

**預期結果**：
- 錯誤測試頁面正確顯示
- 頁面顯示錯誤觸發按鈕
- 點擊按鈕後觸發測試錯誤
- 錯誤被正確捕獲
- 如果整合了 Sentry，錯誤被發送到 Sentry
- 錯誤不會導致應用程式崩潰
- 可以查看錯誤詳情（如果提供）

---

### UX0033: 驗證 Sentry 整合

**觸發條件**：開發環境中，應用程式已整合 Sentry，使用者觸發測試錯誤

**操作步驟**：
1. 開發環境中，應用程式已配置 Sentry
2. 使用者觸發測試錯誤（UX0032）
3. 系統捕獲錯誤並發送到 Sentry
4. 開發者可以在 Sentry 控制台查看錯誤報告

**預期結果**：
- 錯誤被正確捕獲
- 錯誤資訊被發送到 Sentry
- Sentry 控制台顯示錯誤報告
- 錯誤報告包含完整的錯誤堆疊資訊
- 錯誤報告包含相關的上下文資訊（如果配置）
- 可以追蹤錯誤的發生頻率和影響範圍

---

## UI 元素

- 錯誤觸發按鈕
- Sentry 測試功能
- 錯誤詳情顯示（如果提供）

- 測試錯誤捕獲
- 驗證 Sentry 整合

此功能僅在開發環境啟用（由 `constants.ts` 中的 `ENABLE_ERROR_TEST` 控制）。生產環境中此功能不會顯示，確保不會影響一般使用者的體驗。
